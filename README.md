# MOODÉA

스킨케어 및 뷰티 제품을 판매하는 React 기반 웹사이트 프로젝트입니다.  
실제 쇼핑몰과 유사한 레이아웃과 인터랙션을 구현하며,  
UI/UX 설계부터 컴포넌트 구조화, 데이터 연동까지 **실무형 개발 흐름을 경험하기 위해 제작했습니다.**


---

## 🧴 프로젝트 개요
- **컨셉:** 스킨케어 및 뷰티 제품을 판매하는 웹사이트  
- **목적:** 프론트엔드 개발 실무 감각 향상 및 컴포넌트 재활용 구조 학습  
- **기간:** 2025.09 ~ 진행 중  
- **주요 기술:** React, React Hooks, React Router, Vite, Supabase, JavaScript(ES6), CSS3 / Swiper.js

## 컴포넌트 제작 흐름

```

컴포넌트 제작 흐름

1. 컴포넌트 제작소 (특정 전용 컴포넌트 제작)
- 각 페이지/기능에 맞는 전용 컴포넌트 제작
- 예: SkincareList.jsx

2. 제작된 컴포넌트 재활용 (범용성 확보)
- 공통 UI/로직을 추출해서 props 기반 컴포넌트로 리팩터링
- 불필요한 중복 코드를 줄이고, 여러 페이지에서 사용할 수 있도록 수정
- 예: HeaderSectionCategory.jsx
- props: categoryName, categoryBio, jsonFile, arrayName, basePath 등

3. 컴포넌트 대표 페이지에서 데이터 바인딩 (화면 구성)
- 실제 페이지 단에서 데이터 주입
- Supabase DB에서 데이터를 불러와 Recycling 컴포넌트에 props 전달
- 여러 개의 공용 컴포넌트를 조합해서 하나의 완성된 페이지를 구성

```

- 장바구니 페이지 UI 제작 : 장바구니 페이지 UI제작 완료
- 장바구니 기능 구현(1): 비로그인 상태에서 CART 버튼 클릭 시 로그인 필요 안내 모달 노출. 로그인 상태일 경우에만 장바구니 로직 실행.
- 장바구니 기능 구현(2): 장바구니 중복 검사 로직 구현. 이미 동일 상품이 존재할 경우 quantity만 증가시키고, 
없을 경우 insert를 통해 Cart 테이블에 상품 정보 저장.
- 장바구니 기능 구현(3): Cart 테이블 데이터를 불러와 map으로 목록 렌더링.
- 장바구니 기능 구현(4): supabase delete 기능 연결. 삭제 클릭 시 해당 상품 id 조건으로 Cart 테이블 row 제거 후 
state에서도 필터링하여 화면에 즉시 반영.
- 장바구니 기능 구현(5): 최종 결제 금액 모달 "총 금액액, 상품 총 개수, 5만원 이상 시 배송비 0원" reduce메소드를 이용하여 로직 구현
- 장바구니 기능 구현(6): Math.floor 함수를 사용하여 총 금액에서 1000을 나눠 반내림을 함으로써, 1000원당 10포인트 적립 로직 구현
- 장바구니 기능 구현(7): 회원가입 시 신규 회원 쿠폰 발행 기능 구현 
슈퍼베이스 user_coupons테이블에 insert로 쿠폰 지급 구현 후 해당 id를 가진 유저가 보유한 쿠폰 수를 불러와 mypage에 보유 쿠폰 수 데이터 바인딩
- 쿠폰/포인트 사용 모달 : 쿠폰/포인트 사용 버튼 클릭 시 조건부 렌더링을 이용하여 state값을 true로 변환 후 모달창 띄움 + ui 제작
- 쿠폰/포인트 모달 기능 구현(8) : 슈퍼베이스 DB를 기반으로 쿠폰/포인트 보유 수 데이터 바인딩 후 사용할 포인트 인풋창에 숫자만 입력 가능한 정규식 적용
- 쿠폰/포인트 모달 기능 구현(9) : 전체사용 버튼 클릭시 state값에 보유한 포인트 state값을 대입 후 input value에 해당 스테이트 값을 데이터 바인딩 
- 쿠폰/포인트 모달 기능 구현(10) : if문을 사용하여 보유 포인트에 비례하여 “적용” 버튼 클릭 시 간단한 alert 처리
- 쿠폰/포인트 모달 기능 구현(11) : 적용된 할인 정보를 state에 저장해 최종 결제 금액 모달에 실시간 반영하도록 데이터 바인딩
- 장바구니 결제 기능 구현 : 사용자가 장바구니 페이지에서 ‘구매하기’ 버튼을 클릭하면 장바구니에 담긴 전체 상품 정보를 기반으로 주문명 생성, 총 결제 금액(상품 가격 + 배송비 − 쿠폰/포인트 할인)등 계산 후, 
해당 데이터를 payment.js로 전달하여 토스페이먼츠 결제창을 실행하도록 구현
- 장바구니 커스텀훅 제작 : 기존에 있던 길던 장바구니 코드를 cart.js로 만들어 import하여 재사용

하여 해당 view로 닉네임 중복검사 로직을 구현
- 회원정보 수정 로직(2) : input요소를 다 입력하고 정보수정 버튼을 클릭하면 입력된 값을 UPDATE
- 회원탈퇴 UI : 회원탈퇴 UI 완성
- 쿠폰/포인트 바인딩 : 슈퍼베이스에 있는 쿠폰/포인트 수량을 가져와 보유한 수만큼 데이터 바인딩
- 장바구니 바인딩 : 장바구니에 담긴 상품 재활용 컴포넌트 제작하여 props로 마이페이지에 전송 후 데이터 바인딩
- 총 구매금액 표시 기능 : DB에 저장된 결제 금액 데이터를 가져오고, reduce 메서드로 총 금액을 합산하여 마이페이지에 바인딩
- 최근 주문한 상품 데이터 바인딩 : 구매한 상품을 DB에 저장하고, 마이페이지의 최근 주문한 상품 ui에 해당 데이터를 불러와 표시하는 기능 구현
- 최근 주문한 상품 UI 레이아웃 통일 : 장바구니·나의 리뷰 UI와 동일한 구조 및 간격으로 재정비하여 마이페이지 전반의 디자인 일관성을 확보
- 마이페이지 장바구니 정렬 수정 : 정렬 기준을 id → created_at 내림차순으로 변경하여 장바구니 UI가 실제 담긴 시간 기준으로 올바르게 표시되도록 개선
- 마이페이지 프로필 사진 변경 : 슈퍼베이스 스토리지를 이용하여 업로드 된 이미지를 스토리지에서 url로 발급 받은 후 state에 담아 데이터 바인딩

## 결제하기 (로그인 시만 이용 가능)
- 토스페이먼츠 API 연동 & 상품 디테일페이지에서 결제하기 구현 : 사용자가 결제버튼 클릭 시 결제 정보(금액,주문번호 등)를 payment.js로 전달 후, 해당 데이터 기반으로 토스페이먼츠 결제 API를 호출하여 결제창을 실행
- 결제 성공 페이지 UI : 결제 성공 시 보여줄 페이지 UI 제작 완료
- 결제 실패 페이지 UI : 결제 실패 시 보여줄 페이지 UI 제작 완료
- Edge Function API 설정: 구매한 상품 정보를 슈퍼베이스 서버로 안전하게 전송하기 위한 환경 및 Supabase 설정
- 총 구매금액 표시 기능 : API를 통해 DB에 저장된 결제 금액 데이터를 가져오고, 클라이언트에서 fetch GET 요청으로 유저의 결제 금액 목록을 받아온 뒤 reduce 메서드로 총 금액을 합산하여 마이페이지에 바인딩
- 장바구니 결제 기능 구현 : 사용자가 장바구니 페이지에서 ‘구매하기’ 버튼을 클릭하면 장바구니에 담긴 전체 상품 정보를 기반으로 주문명 생성, 총 결제 금액(상품 가격 + 배송비 − 쿠폰/포인트 할인)등 계산 후, 해당 데이터를 payment.js로 전달하여 토스페이먼츠 결제창을 실행하도록 구현
- 결제하기 금액 검증 로직 구현 : 결제 시작 시 DB에 금액을 미리 저장하여 클라이언트 조작 방지, 결제 완료 후 DB 금액 vs Toss 금액 & 클라이언트 금액 vs Toss 금액 두 단계를 검증
- 결제 적립금(포인트) : 구매 시 적립금(포인트)을 OrderHeaders 테이블에 임시 저장한 후, 결제 완료 시 서버에 저장된 포인트 값을 기준으로 사용자 포인트를 업데이트
- 결제 쿠폰 사용 : 구매 시 선택한 쿠폰 ID를 OrderHeaders 테이블에 임시 저장하고, 결제 완료 후 해당 쿠폰을 user_coupons 테이블에서 used = true로 업데이트
- 결제 포인트 사용 : 구매 시 사용 예정 포인트를 OrderHeaders 테이블에 임시 저장하고, 결제 완료 후 [기존 보유 포인트 + 적립 포인트 - 사용 포인트] 를 계산하여 users 테이블의 포인트를 UPDATE
